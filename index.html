    1. Binary Tree (BT)

    A Binary Tree is the most basic tree structure where each node has at most two children, referred to as the left child and the right child. There are no rules about the order of nodes, so values can be placed anywhere. Insertion is simple: we just find the first available spot.
    Example: A family tree can be a binary tree, where a person (node) has at most two children.

    2. Binary Search Tree (BST)

    A Binary Search Tree is a special type of binary tree with a key property: for any given node, all values in its left subtree are less than the node's value, and all values in its right subtree are greater. This ordering makes searching for a value highly efficient, typically in O(log n) time on average.
    Example: If you insert 50, then 30, 30 goes to the left of 50. If you then insert 70, it goes to the right of 50.

    3. AVL Tree

    An AVL Tree is a self-balancing Binary Search Tree. It maintains the BST property but adds a constraint: the heights of the two child subtrees of any node can differ by at most one. If an insertion or deletion violates this rule, the tree automatically rebalances itself using "rotations." This guarantees that operations remain extremely fast (O(log n) in the worst case), preventing the tree from becoming skewed.
    Example: If you insert 10, 20, and 30 in order into a BST, you get a straight line. An AVL tree would rotate after inserting 30 to make 20 the root, with 10 as its left child and 30 as its right.
